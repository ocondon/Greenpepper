Statistics & Annotations

Statistic are compiled when document are executed.
At the end of the execution you will generally receive a summary of these statistic

Example
Running GreenPepper Some Specification Name
20 tests: 17 right, 2 wrong, 1 ignored, 0 exception(s)
Begin Info

There are 4 different counters in the statistic

  • Right
  • Wrong
  • Ignored
  • Exception

Right and wrong are trivial. They are place on cells where there are expectations depending if it was meet or not.

Ignored are place on cells where there should have been an expectation, but the cell was left blank.
Ignored can also happens when a entire table was commented out.

Exceptions can occurs for various reason. Often the are sign of a problem between the example and the fixture. The name of the test does not match the name of any method in the fixture. When an exception occurs, you should be aware that the other results in the tests might be corrupted, and the exception should be corrected before given any value to the rights and wrongs of the specification.

End Info

Rule For interpreter

Each cell that is part of a column with a test (column header that ends with ? or () ) will be annotated and will increments statistic for Right wrong, ignored or exception.

[9][3][ignored]
[6][0][0 exception] 		1 1 1 1

Exceptions on column headers will count as one exception. The column header will be annotated with an exception and the exception counter will be incremented by one. From the moments there is a exception in a column header of a rule for, the specification does not prescribe the annotations and or statistic for the rest of the column.
